# Editor

DocForge includes a built-in web-based editor for reviewing, refining, and exporting generated documents. The editor loads after generation and preserves the formatting from your original template.

## Overview

The editor provides:

- WYSIWYG text editing with formatting preservation
- Section-by-section review of generated content
- LLM-generated content highlighting for easy identification
- Low-confidence item flagging for targeted review
- Per-section LLM regeneration with prompt modification
- Table editing capabilities
- Export to `.docx` with original template formatting

## Opening the Editor

After a successful generation, the editor opens automatically in the web UI. You can also open a previous generation's document:

1. Go to the project workspace.
2. Open the **Generation History** panel.
3. Click on a generation run to open it in the editor.

Via the API, retrieve the editor document:

```bash
curl http://localhost:8000/api/v1/generations/{run_id}/document
```

This returns the document in TipTap-compatible JSON format. If a user has made edits, the saved editor state is returned. Otherwise, the output `.docx` is converted on-the-fly and cached.

## Reviewing Content

### LLM-Generated Sections

Sections generated by the LLM are visually highlighted in the editor. Review these sections to verify the content meets your requirements.

### Low-Confidence Items

Items that were auto-resolved with medium confidence (0.4-0.79) are flagged with warning indicators. These warrant closer review to confirm the data mapping was correct.

### Render Errors

If any markers failed to render during generation, they appear with their original red text or an error marker. Check the generation report for details on what went wrong.

## Editing

The editor supports standard text editing operations:

- Edit text content directly
- Modify table cell values
- Adjust formatting (bold, italic, etc.)
- All changes are tracked and can be saved

### Saving Edits

Edits are saved to the database and persist across sessions:

```bash
curl -X PUT http://localhost:8000/api/v1/generations/{run_id}/document \
  -H "Content-Type: application/json" \
  -d '{ ... editor state JSON ... }'
```

## Regenerating Sections

If you are not satisfied with an LLM-generated section, you can regenerate it without re-running the entire document:

1. Click the **Regenerate** button on the section.
2. Optionally modify the prompt to refine the instruction.
3. The LLM generates new content, which replaces the section in the editor.

Via the API:

```bash
curl -X POST http://localhost:8000/api/v1/generations/{run_id}/regenerate-section \
  -H "Content-Type: application/json" \
  -d '{
    "markerId": "marker_2",
    "modifiedPrompt": "Provide a more concise summary focusing on revenue growth"
  }'
```

Response:

```json
{
  "content": "Revenue grew by 15% year-over-year...",
  "llmUsage": {
    "model": "gpt-4o",
    "prompt_tokens": 450,
    "completion_tokens": 120
  }
}
```

The frontend patches the new content into the editor state. This does not create a new generation run.

## Exporting

When you are satisfied with the document, export it as a `.docx` file:

1. Click **Export** in the editor toolbar.
2. The editor state is converted back to a `.docx` file with the original template formatting preserved.
3. The file downloads to your browser.

Via the API:

```bash
curl -O http://localhost:8000/api/v1/generations/{run_id}/export
```

If no edits have been made in the editor, the original generated `.docx` is returned. If edits exist, the editor state is converted back to `.docx` using the original template as a formatting reference.

## Editor State Format

The editor uses a TipTap-compatible JSON format internally. The `EditorDocument` model captures:

- Document content (paragraphs, tables, headings)
- Metadata from the generation run
- Marker annotations for LLM-generated content
- Confidence scores for auto-resolved items

This state is stored in the `editor_state` column of the `generation_runs` table and is updated whenever the user saves their edits.
